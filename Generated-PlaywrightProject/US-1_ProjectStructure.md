```
e2e/                                 # Root directory for the Playwright end-to-end test suite.
├── src/                             # Contains the core components and building blocks of the automation framework.
│   ├── pages/                       # Stores Page Object Model (POM) classes, abstracting UI interactions and element locators.
│   │   ├── common/                  # Contains base page classes or elements common across multiple pages.
│   │   └── authentication/          # Page objects specific to authentication flows, like login and logout.
│   │       └── LoginPage.ts         # Page object for the application's login page.
│   ├── fixtures/                    # Custom Playwright fixtures for setting up test environments, authenticated sessions, or data.
│   │   └── auth.fixture.ts          # An example fixture providing authenticated user contexts for tests.
│   ├── utils/                       # General utility functions for various non-UI tasks such as logging, date formatting, or file operations.
│   │   └── logger.util.ts           # Utility for consistent logging throughout the framework.
│   ├── helpers/                     # Contains reusable helper functions for common actions or assertions across tests, not tied to a specific page.
│   │   └── ui.helper.ts             # Helper functions for common UI interactions like waiting, clicking, or verifying elements.
│   ├── constants/                   # Stores application-wide constants such as URLs, API endpoints, timeouts, or specific string values.
│   │   └── app.constants.ts         # Global application constants, including base URLs and environment details.
│   └── types/                       # TypeScript declaration files for custom types and interfaces used within the framework.
│       └── user.types.ts            # Defines types for user objects, including credentials and roles.
├── tests/                           # Contains the actual test specifications, organized by feature, module, or test type.
│   ├── authentication/              # Test suites dedicated to validating authentication and login functionalities.
│   │   └── login.spec.ts            # Test file containing scenarios for salesman login, including success and failure cases.
│   └── order-management/            # Test suites for features related to creating, viewing, and managing orders.
├── config/                          # Stores Playwright configuration files and environment-specific settings.
│   ├── playwright.config.ts         # The main Playwright configuration file, defining projects, reporters, and global setup.
│   └── environments/                # Directory for environment-specific configuration variables (e.g., dev, qa, prod URLs).
│       ├── dev.ts                   # Configuration settings for the development environment.
│       └── qa.ts                    # Configuration settings for the QA (Quality Assurance) environment.
├── test-data/                       # External data files used by tests, such as user credentials, input values, or expected outputs.
│   ├── users.json                   # JSON file containing various user credentials for different roles, including salesmen.
│   └── salesman-login-data.json     # Specific test data for the salesman login scenarios, e.g., valid and invalid credentials.
├── reports/                         # Output directory for test execution reports, screenshots, and videos generated by Playwright.
│   ├── html-report/                 # Contains the interactive HTML report generated after test runs.
│   ├── screenshots/                 # Stores screenshots captured during test failures for debugging.
│   └── videos/                      # Stores video recordings of test execution for visual debugging.
├── package.json                     # Project manifest file, listing dependencies, scripts, and project metadata.
├── tsconfig.json                    # TypeScript configuration file for the project, defining compiler options.
└── .env                             # Environment variables for local development or CI/CD pipelines, storing sensitive data.
```
