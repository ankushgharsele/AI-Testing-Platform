## GitHub Personal Access Token (PAT) Management - Test Plan

**User Story:** As a GitHub user, I want to be able to create and manage my personal access tokens (PATs) securely so that I can authenticate with GitHub API and third-party applications.

**Assumed Acceptance Criteria:**
*   Users can navigate to the "Developer settings" section within their profile.
*   Users can access a dedicated "Personal Access Tokens" page.
*   Users can generate a new PAT by providing a descriptive name (note).
*   Users must select one or more scopes/permissions for the PAT.
*   Users can optionally set an expiration date for the PAT (e.g., 7 days, 30 days, 90 days, no expiration).
*   Upon successful creation, the PAT string is displayed *once* to the user, with clear instructions to copy it.
*   Users can view a list of their existing PATs, including their name, scopes, creation date, and expiration status.
*   Users can revoke an existing PAT from the list.
*   The PAT string is never displayed again after initial creation.
*   PATs created via the UI should correctly authenticate with GitHub API based on their assigned scopes.
